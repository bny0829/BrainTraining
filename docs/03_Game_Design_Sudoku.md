# 03 數獨玩法規格

*更新：2026-07-12（v0.1）*

## 一、規則與操作

- 9×9 標準數獨，題目保證**唯一解**。
- 點選格子 → 高亮同列／同行／同宮（淡藍）與同數字（較深藍）。
- 數字鍵 1–9 輸入；某數字已正確放滿 9 格時該鍵停用。
- **錯誤上限 3 次**：填錯立即顯示紅色並計數，達上限跳「挑戰失敗」（可重新開始同一題）。
- **筆記模式**：切換後輸入數字＝在格內加／移除小字候選數。正確填入某數字時，自動清除同列／行／宮筆記中的該數字。
- **提示**：直接填入所選格的正確答案（未選格子時自動挑第一個未完成格），次數記錄在結算。
- **復原**：單格層級的操作復原（最多 200 步），不會回復錯誤計數（防止作弊）。
- **自動存檔**：每一步即存，可從首頁「繼續」無縫續玩。

## 二、難度規格

| 難度 | 目標提示數 | 每日輪替 |
| --- | --- | --- |
| 簡單 ★ | 42 | 週一 |
| 普通 ★★ | 34 | 週日、二、三 |
| 困難 ★★★ | 29 | 週四、五 |
| 專家 ★★★★ | 26 | 週六 |

### 產生器（`SudokuLogic.generate`）

1. 隨機回溯填出完整合法盤面（MRV 加速）
2. 中心對稱成對挖空，每挖一格用解題器驗證仍是唯一解
3. 對稱挖不動時改單格挖空，逼近目標提示數

實測（Godot 4.7 headless）：簡單 11ms、專家 316ms，可即時產生。

### 難度評估器（`SudokuLogic.rate`）— v0.1 基本版

模擬人類基本技巧（唯一候選數、隱性唯一數）解題：只需基本技巧 → 簡單／普通；卡住需進階技巧 → 困難／專家。

**已知限制**：目前以提示數控制難度，低提示數的題目仍可能被基本技巧解完（評估器會回報 MEDIUM）。v0.2+ 改進方向：產生後用 `rate()` 驗收，不符難度就重生成；並擴充技巧庫（區塊摒除、數對、X-Wing）讓分級更準。

## 三、每日挑戰

- 種子 = `yyyymmdd` 整數 → 同一天全球同一題（離線、零成本）。
- 難度依星期輪替（見上表），營造「週六 Boss 題」節奏。
- 完成 → streak +1；中斷一天歸零。首頁顯示「連續挑戰 N 天」。
- 當日未完成的每日挑戰進度跨日即作廢。

## 四、結算與統計

- 勝利：記錄該難度最佳時間、完成局數；每日挑戰另更新 streak。
- 失敗（3 錯）：計入場次，不計最佳時間。
- 首頁戰績卡：完成局數／總局數＋各難度最佳時間。
