# 05 五子棋玩法規格

*更新：2026-07-12（v0.2）*

## 一、規則與操作

- 15×15 標準棋盤，先連五者勝（無禁手，休閒取向）。
- 玩家執黑先手，AI 執白。點擊交叉點落子，最後一手有橙色標記。
- **悔棋**：一次收回 AI 與玩家各一手（AI 思考中與對局結束時停用）。
- **重新開始**：確認對話框後棄局重開（同難度）。
- **自動存檔**：每手即存（存落子序列），可從首頁「繼續」還原；還原＝重播序列。
- 對局結束（勝/負/平）記入 `gomoku_stats`：場次、勝場、各難度勝場。

## 二、AI 設計（`GomokuLogic`）

決策流程（`choose_move`）：

```
1. 有即勝點 → 直接下
2. 對手有即勝點 → 擋（初學級 30% 機率漏看）
3. 候選手排序（進攻分 + 0.9 × 防守分）→ 依難度搜尋
```

| 難度 | 策略 | 實測速度 |
| --- | --- | --- |
| 初學 ★ | 不搜尋，前 3 熱點隨機挑一、會漏擋 | 即時 |
| 普通 ★★ | Negamax 深度 1（貪婪＋全盤評估） | 即時 |
| 困難 ★★★ | 深度 2、候選寬度 10 | ~15 ms/手 |
| 專家 ★★★★ | 深度 3、候選寬度 8 | ~170 ms/手 |

技術要點：

- **候選手剪枝**：只考慮既有棋子 2 格內的空點，空盤下中央。
- **評估函數**：掃描全部 5 連窗口（572 個），單色窗口依棋子數計分，混色窗口無價值；對手威脅權重 ×1.1（偏防守）。
- **型態分數**（`point_score`）：連五 > 活四 > 沖四 > 活三 > 眠三…，用於候選排序與淺層決策。
- **背景執行緒**：`choose_move` 為純函式，畫面用 `Thread` 執行、`_process` 輪詢結果，UI 不凍結；離開畫面時必須 `wait_to_finish()`。

此「候選手 + 型態分數 + Negamax」框架即 docs/02 所說的共用 AI 模組雛形，黑白棋（v0.3）將重用同樣結構（換掉走子規則與評估函數）。
