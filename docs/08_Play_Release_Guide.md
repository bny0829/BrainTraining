## 四、Play Console 表單答案

**資料安全（Data safety）**——全部照實填「不收集」：
- 是否收集或分享使用者資料？ → **否**
- 資料是否加密傳輸？ → 不適用（無網路功能）
- 使用者能否要求刪除資料？ → 不適用

**內容分級問卷**：益智遊戲、無暴力/賭博/色情/使用者互動 → 會拿到全年齡（3+）。

**廣告**：本 App 不含廣告 → 否。

**目標對象**：13 歲以上（選含兒童會多很多審查要求，先避開）。

## 五、匯出 AAB（Play 只收 AAB，不收 APK）

1. Godot 選單：**專案 → 安裝 Android 建置樣板…** → 確認（一次性；會生成 `android/` 資料夾，已加入 .gitignore）。沒做這步匯出視窗會顯示紅字「尚未在專案中安裝 Android 建置樣板」
2. 專案 → 匯出 → Android 預設組：
   - `Gradle 構建 / 使用 Gradle 構建` 打勾
   - `匯出格式` 選 **Export AAB**
   - Export Path 改成 `./BrainClub.aab`
3. 確認已重啟過 Godot（讀取金鑰環境變數），**不要**勾「以除錯模式匯出」
4. 按「匯出專案」。**第一次 Gradle 建置需下載相依套件（需網路，約 3～10 分鐘）**，輸出面板會持續顯示進度

版號規則：每次上傳 Play 的 `version/code` 必須**比前一次上傳大**（目前 7），跳號完全沒關係（1~6 沒上傳過不影響）；`version/name` 對玩家顯示（目前 0.7.0）。封測從 v0.7 開始，測試者直接拿到六款遊戲完整版。

## 六、封閉測試（個人帳號硬性要求）

1. Play Console → 建立應用程式（名稱 Brain Club、遊戲、免費）
2. 左側「測試 > 封閉測試」→ 建立測試版本 → 上傳 AAB → 填寫版本資訊
3. 建立測試人員名單：**至少 12 人**（親友的 Gmail 都可以），分享加入連結請他們接受並安裝
4. 維持 **14 天**不中斷（期間可以更新版本）
5. 14 天後 Console 會開放「申請正式版」→ 填寫上架審查表單 → 送審（通常數天）

**現在就可以做的事**：帳號驗證一過，立刻上傳封測並找齊 12 位測試者——14 天的時鐘越早開始越好。期間我們繼續開發 v0.6。
